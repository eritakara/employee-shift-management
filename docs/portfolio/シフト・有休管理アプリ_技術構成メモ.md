# シフト・有休管理アプリ 技術構成メモ

本アプリケーションは、Java Servlet/JSP の堅牢なバックエンドを基盤とし、ローカル開発環境（H2 データベース）と本番環境（Supabase PostgreSQL）の両方でシームレスに動作するポータブルな Web アプリケーションです。インフラは Docker を用いて Render 上にコンテナ展開しています。

---

## 1. システムアーキテクチャ概要

```mermaid
flowchart TD

    SqlCompat <-->|JDBC (DriverManager / SSL)| Supa
    SqlCompat <-->|JDBC (H2 File)| H2
```

## 3. SQL 互換レイヤー (H2 ⇄ PostgreSQL) の設計

開発時に用いる H2 データベースと本番の PostgreSQL では、SQL の組み込み関数に差異があります。本アプリでは、Java コード側で `Database.isPostgres()` メソッドを用いてクエリを動的に置換、あるいは条件分岐することで対応しています。

### 代表的な置換と分岐例

#### ① 日付の加算・減算
* **H2 (ローカル)**: `DATEADD('day', 1, work_date)`
* **PostgreSQL (本番)**: `(work_date + INTERVAL '1 day')::date`
* **実装方式**: SQL文字列を `isPostgres()` の判定に基づき、Java 内で `work_date + INTERVAL '1 day'` へ文字列置換またはクエリ分岐してバインドします。

#### ② 日付の文字列フォーマット
* **H2 (ローカル)**: `FORMATDATETIME(work_date, 'yyyy-MM')`
* **PostgreSQL (本番)**: `TO_CHAR(work_date, 'YYYY-MM')`
* **実装方式**: `DashboardService.java` 等において、DB 種別を判定した上で SQL 文字列を動的に構築します。

#### ③ 2日間の時間差（秒数）の取得
* **H2 (ローカル)**: `DATEDIFF('second', clock_in, clock_out)`
* **PostgreSQL (本番)**: `EXTRACT(EPOCH FROM (clock_out - clock_in))`
* **実装方式**: 勤怠集計クエリ部分で SQL の分岐を行い、集計ロジックの正確性を両立しました。

## 4. 環境変数設計とセキュリティ安全装置

秘密情報の漏洩や本番環境での脆弱性侵入を防ぐため、OS の環境変数に基づいた安全装置（防衛的プログラミング）を設計・導入しています。

### 環境変数一覧

| 環境変数名 | 想定される値 | 用途と動作 |
|---|---|---|
| `APP_ENV` | `production` / `development` | 動作モード。`production` の場合、エラー画面での例外詳細出力をマスクします。 |
| `JDBC_URL` | `jdbc:postgresql://<Supa-Host>:5432/...` | 本番 PostgreSQL (Supabase Connection Pooler) 接続用 URL。 |
| `DB_USER` | `postgres.<project-ref>` | 本番 DB 接続ユーザー。 |
| `DB_PASSWORD` | `<Supabase-Password>` | 本番 DB 接続パスワード。 |
| `INITIAL_HR_EMAIL` | `admin@example.com` | 本番に初期登録される最初の人事管理者のメールアドレス。 |
| `INITIAL_HR_PASSWORD` | `<強力な任意のパスワード>` | 初期管理者のパスワード。デフォルト値（`Password1!`）のまま起動しようとすると安全のため起動例外を発生させて停止します。 |
| `DEMO_ACCOUNTS_ENABLED` | `true` / `false` | ポートフォリオデモ環境用の全拠点デモアカウント（35名）を自動生成するかどうかのフラグ。 |
| `DEMO_ACCOUNTS_PASSWORD` | `<強力な任意のパスワード>` | 生成されるデモアカウントの共通パスワード。本番環境で `DEMO_ACCOUNTS_ENABLED=true` の場合に必須。デフォルト値（`Password1!`）のまま起動しようとすると起動停止します。 |

### 本番セキュリティガード (安全装置)
1. **初期パスワードチェック**: 
   本番稼動環境で初期管理者 (`hr@example.com`) またはデモアカウントのパスワードが `Password1!`（デフォルト値）のままであったり空文字の場合は、`IllegalStateException` をスローして Tomcat の起動をブロックします。
2. **テーブル名ホワイトリスト検証**: 
   マスターIDを動的に取得するヘルパーメソッド `findIdByName` にて、外部からの SQL インジェクションを防ぐため、引き渡し可能なテーブル名を `"branches"`, `"departments"`, `"employment_types"` に限定するホワイトリストチェックを Java コードの最前線で実行します。
