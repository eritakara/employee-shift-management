# ShiftFlow 要件定義書

- 文書版: 2026-06-19
- 対象: 現在の `main` 作業ツリー（今回の画面・機能・デモデータ変更を含む）
- 関連資料: `TASK_LIST.md`、`UAT_TEST_CASES.md`、`SCREEN_WIREFRAMES.md`

## 1. 役割と表示範囲

| 役割 | ダッシュボード | シフト画面 | 更新権限 |
|---|---|---|---|
| 従業員 | 本人を対象とする指標・勤務情報 | 本人の月間シフト | 希望シフト、変更・休み申請など本人向け操作 |
| 店長 | 指標と支店別月間シフト | 担当範囲のシフト | シフト調整・確定、申請承認など店長向け操作 |
| 人事担当者 | 全社指標と支店別月間シフト | 全社管理対象のシフト | マスター、従業員、全社確認など人事向け操作 |

- 未認証時は認証フィルターでログイン画面へ遷移する。
- 権限外画面はHTTP 403、存在しない画面はHTTP 404を返す。
- ダッシュボードの支店プルダウンは、シフトデータが存在する有効支店を表示する。

## 2. 画面要件

### 2.1 共通メニュー

- 「概要」配下は「通知」「ダッシュボード」の順に表示する。
- 本人用シフト画面の表記は「シフト」とする。
- 日本語・英語表示を切り替えられる。英語時の本人用シフト表記は `My schedule` とする。

### 2.2 ダッシュボード

- 画面上部に当月残業時間と45時間基準の残業アラートを表示する。
- 36時間未満を安全、36時間以上45時間未満を注意、45時間以上を警告として色と文言を切り替える。
- 今日の勤務者、未承認申請、有休残日数、人員不足、今月の実勤務を表示する。
- 有休残日数は不要な小数末尾を付けず、例として `10日` と表示する。
- 人員不足は日勤・夜勤それぞれについて、必要人数に対する不足割合を百分率で表示する。不足がない場合は0%とする。
- 「今月のシフト」は月間表形式とし、氏名・社員番号を行、日付を列として表示する。
- 表の左端に連番を表示し、右端に日勤・夜勤・夜勤明け・休み・有休の月間回数を表示する。
- 勤務区分は日勤、夜勤、夜勤明け、休み、有休、半休で色分けし、各セルは「日」「夜」「明」「休」「有」「半」で表示する。
- 半休は「半」の下に「(午前)」または「(午後)」を表示する。
- 店長・人事担当者は支店プルダウンを変更すると、表示ボタンなしで対象支店の表へ切り替えられる。
- 対象支店・対象月にシフトがない場合は「対象月のシフトはありません。」と表示する。

### 2.3 シフト

- `/app/shifts/mine` のページタイトル、見出し、メニュー表記は「シフト」とする。
- 対象月入力は `YYYY-MM` の月選択とし、変更時に表示ボタンなしで自動送信する。
- 選択した対象月の本人の月間シフトを、ダッシュボードと同じ勤務区分・色・略称で表示する。
- 「印刷」から印刷画面へ遷移し、「印刷する」でブラウザ印刷を起動する。
- 印刷画面には「シフトへ戻る」を表示し、選択月を維持して本人用シフト画面へ戻る。
- 対象月にシフトがない場合は「対象月のシフトはありません。」と表示する。

## 3. 入力項目

| 画面 | 項目 | 形式・動作 |
|---|---|---|
| ダッシュボード | 支店 | プルダウン。変更時に自動送信。選択値は `branchId` |
| シフト | 対象月 | 月入力。変更時に自動送信。選択値は `month` |
| 印刷 | 印刷する | ボタン押下でブラウザ印刷を実行 |
| 印刷 | シフトへ戻る | 選択月をクエリに保持して戻る |

- 不正または未指定の月はサーバー側で当月として扱う。
- 存在しない支店IDは、シフトが存在する先頭支店へ補正する。

## 4. データ要件

- 開発起動時のみ、システムプロパティ `shiftapp.seedDemoShifts=true` によりデモCSVを取り込む。
- `demo-users.csv` は社員番号、氏名、メール、役割、支店、部署、雇用形態を保持する。
- `demo-shifts.csv` は社員番号、対象月、勤務パターン、有休日、半休日、半休区分、状態、備考を保持する。
- 2026年6月分は17名を那覇支店6名、北部支店6名、中部支店5名に配属する。
- 各支店で毎日、日勤と夜勤を最低1名ずつ配置する。
- 夜勤の翌日は夜勤明けとし、有休は各人月1回、半休は午前・午後を混在させる。
- 勤務区分 `NIGHT_OFF`（夜勤明け）を初期データに追加する。
- CSV取込は社員番号および利用者・勤務日の組み合わせをキーとして再実行可能とし、既存利用者のパスワードは上書きしない。
- デモ利用者には有休残数10日を設定する。
- CSVヘッダー、列数またはリソース不足がある場合は起動処理を失敗させ、原因をログに残す。

## 5. エラー・状態表示

- 更新処理成功時はフラッシュメッセージ、業務例外・入力不正時はエラーバナーを表示する。
- 確定前の人員不足や夜勤後休息違反は警告一覧に表示し、警告付き確定では理由を必須とする。
- 月間シフトが0件の場合は空状態メッセージを表内に表示する。
- 残業基準到達時は `alert`、基準未到達時は状態通知として支援技術へ伝える。

## 6. 変更前資料との差分

| 分類 | 変更前 | 最新仕様 |
|---|---|---|
| 本人用シフト名称 | 自分のシフト／Myシフト | シフト |
| ダッシュボード予定 | 最大7件の縦一覧 | 支店別の月間シフト表 |
| 勤務状態 | 状態文字列を併記 | 色分けした略称と月間集計 |
| 半休 | 半休のみ | 午前・午後を明記 |
| 人員不足 | 不足区分数 | 日勤・夜勤別の不足率 |
| 残業 | 指標のみ | 45時間基準の3段階アラート |
| 有休残数 | 小数2桁 | 不要な小数を除いた日数 |
| 対象月 | 表示ボタンで更新 | 本人用シフトでは変更時に自動更新 |
| 印刷 | 印刷表示 | 印刷、印刷実行、シフトへ戻る |
| デモデータ | 基本アカウント | 3支店17名と2026年6月の勤務データ |
