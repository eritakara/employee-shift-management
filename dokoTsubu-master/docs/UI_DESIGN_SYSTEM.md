# シフト・有休管理 UI・レスポンシブ設計

## 1. 設計原則

- 従業員、店長、人事担当者が同じ画面骨格を使い、権限に応じて必要なメニューと操作だけを表示する。
- 状態は色だけで示さず、必ず文字を併記する。
- 入力、一覧、検索、空状態、エラーの配置と操作順を全画面で統一する。
- PC、タブレット、スマートフォンで機能を削らず、狭い画面では段組みとナビゲーション方式を変える。
- 日本語と英語の文字数差を吸収し、固定幅のテキスト領域を作らない。
- 個人情報、有休理由、位置情報は必要な役割・画面にだけ表示する。

## 2. PCレイアウト（DES-003）

```text
┌──────────────┬─────────────────────────────────────────┐
│              │ 上部バー: ページ名 / 通知 / 利用者 / 設定 / ログアウト │
│ サイドナビ   ├─────────────────────────────────────────┤
│ 248px固定    │                                         │
│              │ コンテンツ（最大1500px、左右余白28px）   │
│ 役割別項目   │                                         │
│              │                                         │
└──────────────┴─────────────────────────────────────────┘
```

- サイドナビは幅248px、画面上端に固定し、項目が多い場合はナビ内を縦スクロールする。
- 上部バーはコンテンツ列上端に固定し、ページ名と利用者情報を常に確認できるようにする。
- コンテンツは可変幅とし、一覧表はページ全体ではなく表領域内で横スクロールする。
- ダッシュボード指標は5列を基本とし、1050px以下で2列へ切り替える。
- 入力フォームは3列を基本とし、1050px以下で2列へ切り替える。

## 3. スマートフォンレイアウト（DES-004）

```text
┌─────────────────────────┐
│ ☰  ページ名       通知 / 設定 │
├─────────────────────────┤
│                         │
│ コンテンツ（左右12px）    │
│ フォーム1列              │
│ 表は領域内横スクロール    │
│                         │
├─────────────────────────┤
│ プライバシーリンク        │
└─────────────────────────┘

☰を押すと幅270pxのナビが左から開く。
```

- 760px以下をスマートフォン幅とする。
- サイドナビは通常非表示とし、メニューボタンで左から表示する。
- 利用者名・所属表示を省略するが、アカウント設定とログアウト操作は維持する。
- フォームは1列、打刻ボタンは縦並び・全幅とする。
- ダッシュボード指標は2列とし、狭い端末でも数値を比較できるようにする。
- タップ対象は原則40px以上、打刻ボタンは54px以上とする。

## 4. 共通コンポーネント（DES-005）

### 入力

| 要素 | 共通ルール |
|---|---|
| ラベル | 入力の上に配置し、必須項目はHTMLの`required`を設定する |
| テキスト | 高さ42px以上、長文は`textarea`、サーバー側でも長さ・形式を検証する |
| 日付・月 | `date`、`month`、`datetime-local`を用途別に使う |
| 選択 | マスター値・権限範囲内の候補だけを表示する |
| 主操作 | ティール背景の`primary`、取消・却下は赤系の`danger-button` |
| 無効操作 | `disabled`と説明文を併用し、色だけで理由を示さない |

### 一覧

- 見出し行は淡いグレー、セルは左揃え、状態は文字付きバッジで表示する。
- 列が収まらない場合は`.table-wrap`内を横スクロールし、本文全体の横スクロールを避ける。
- 0件時は表を消さず、列幅全体に「対象データはありません」と表示する。
- 操作ボタンは右端にまとめ、表示・編集・承認・却下の順を画面間で揃える。

### 検索・絞込み

- 検索条件は一覧上部の独立セクションまたはツールバーへ置く。
- 対象月は画面間で`YYYY-MM`を維持する。
- 条件適用ボタンと「条件をクリア」を並べ、現在条件を入力欄へ保持する。
- 権限範囲外を検索候補へ出さず、URLパラメーターを直接変更してもサーバー側で拒否する。

### ページング・件数上限

- 通常一覧は業務上の月・所属で件数を限定する。
- 監査ログなど大量化する一覧は上限件数を明示する（現行は最大300件）。
- ページング追加時は「前へ／次へ」、現在ページ、総件数を一覧の上下に同じ順序で置き、検索条件を引き継ぐ。

### 確認とエラー

- 破壊的操作は操作名と対象をボタン上で明示し、実装する確認画面・ダイアログでも同じ文言を使う。
- 入力エラーはフォーム上部の赤系アラートと対象入力のブラウザ検証を併用する。
- 保存成功は緑系アラートで1回表示し、再読込で二重登録しないPOST-Redirect-GETを使う。
- 権限エラーは操作を実行せず、個人情報を含まない一般的なメッセージを表示する。
- サーバーエラーは内部SQL、パス、秘密情報を画面へ出さず、ログ側の追跡情報で調査する。

## 5. 状態・警告表示（DES-006）

### 申請・確定状態

| 状態 | 日本語表示 | 色・表現 | 操作 |
|---|---|---|---|
| DRAFT | 下書き | 黄系バッジ | 編集可能 |
| SUBMITTED | 提出済み | 黄系バッジ | 店長確認待ち |
| PENDING | 申請中 | 黄系バッジ | 承認・却下待ち |
| CONFIRMED | 確定 | 緑系バッジ | 通常編集不可 |
| APPROVED | 承認 | 緑系バッジ | 反映済み |
| REJECTED | 却下 | 赤系バッジ | 理由・履歴を確認 |
| CANCELLED | 取消 | グレー系バッジ | 操作終了 |
| COMPLETE | 完了 | 緑系バッジ | 勤怠の出退勤入力済み |
| OPEN | 未完了 | 黄系バッジ | 打刻漏れを確認 |

### メール状態

| 状態 | 意味 | 管理者操作 |
|---|---|---|
| QUEUED | 送信待ち | 待機 |
| SENDING | 送信処理中 | 重複操作しない |
| RETRY | 自動再試行待ち | SMTP状態を確認 |
| FAILED | 最大試行回数到達 | 復旧後に再送 |
| SENT | 送信完了 | 対応不要 |

### シフト警告

- `STAFF_SHORTAGE`: 必要人数に対する人員不足。必要数・実績数・日付・勤務区分を表示する。
- `NIGHT_REST`: 夜勤明け休息期間への勤務登録。対象従業員・日付を表示する。
- 警告は赤文字と説明を併用する。確定を許可する場合も警告を消さず、理由入力を必須にする。
- 警告理由へ個人情報、有休理由、位置情報を書かせない。

## 6. 日英レスポンシブ設計（DES-007）

- ナビゲーション、ボタン、見出し、空状態、エラー、メール、帳票を翻訳対象とする。
- 言語は利用者アカウントへ保存し、次回ログイン後も維持する。
- ボタン・ナビ項目に固定文字幅を設定せず、折返しまたはコンテナ拡張を許可する。
- 英語は日本語より長くなる前提で、操作列は`.actions`による折返しを使う。
- 表のデータ列は不自然に折り返さず、横スクロールを使用する。
- 日付、数値、状態コードは内部値を維持し、表示ラベルだけを翻訳する。
- 日本語フォントを優先し、利用不可時はシステムUIフォントへフォールバックする。
- 760pxと1050pxの境界前後で日英双方を確認し、文字切れがある場合は個別固定幅ではなく共通レイアウトを修正する。

## 7. アクセシビリティ設計

- すべてのフォーム要素に可視ラベルを付ける。
- キーボードフォーカスを3pxの輪郭で表示する。
- ナビゲーションに`aria-label`、アイコンのみの操作にアクセシブル名を付ける。
- 色だけに依存せず、状態文字、警告文、ボタン文言を併記する。
- 表は`thead`と見出しセルを使用し、読み上げ順とDOM順を一致させる。
- スマートフォンでも拡大を禁止せず、ブラウザの文字拡大を許可する。

## 8. 印刷

- 印刷時はサイドナビ、上部バー、入力・操作部、通知アラートを非表示にする。
- コンテンツを全幅へ広げ、表の文字と余白を印刷向けに縮小する。
- 印刷物には対象月と出力対象が分かる見出しを残す。
