# テスト仕様・報告書

本ドキュメントは、本アプリケーションにおける品質検証のテスト方針、実施したテスト項目、および本番環境デプロイ後の動作確認手順についてまとめたものです。

---

## 1. テスト方針

本システムでは、外部のテストフレームワーク（JUnitなど）への依存を最小限に抑え、標準機能を用いた自動スモークテストスクリプト（`test.ps1` および `test-postgres.ps1`）を導入して品質を検証する方針をとっています。

### 主なテスト手法
* **自動ビジネスロジックテスト**:
  * データベースのモックまたは組み込みDB（H2）を使用して、打刻処理や権限チェックのビジネスロジックを検証するテストコードを実行します。
* **手動ブラウザ疎通テスト**:
  * 開発用ローカル環境および本番環境において、ブラウザを用いて実際の画面表示と操作の流れ（ユーザーシナリオ）を確認します。

---

## 2. 確認した主な機能テスト項目

主要な機能要件に対して、以下の観点でテストを実施し、合格判定を行っています。

### 2.1 ログインおよびアクセス制限（権限別テスト）
* **未ログイン状態の防御**:
  * セッションが空の状態で直接 `/app/dashboard` や `/app/attendance/clock` などのURLにアクセスした際、ログイン画面（`/index.jsp`）に正しくリダイレクトされることを確認。
* **権限外画面へのアクセス拒否 (403 Forbidden)**:
  * 一般従業員アカウントでログイン後、店長専用画面（`/app/shifts/manage`、`/app/attendance/manage`）や人事専用画面（`/app/employees`）のURLを直接入力した際、アクセスが拒否されエラー画面が表示されることを確認。
  * 同様に、店長アカウントで人事専用の画面にアクセスした際にもアクセス制限が働くことを確認。

### 2.2 出退勤打刻データの登録検証テスト
* **打刻時刻の記録**:
  * 従業員アカウントで打刻画面を開き、「出勤」および「退勤」ボタンを押した際に、現在時刻が秒単位で `attendance` テーブルに正しく登録されることを確認。
* **打刻状態の自動判定**:
  * 登録された出勤・退勤時刻が、従業員に割り当てられたシフトの開始・終了時間と照合され、遅刻や早退が自動的にフラグ（`late` / `early`）として記録されることを確認。
* **二重打刻の防止**:
  * すでに「出勤」を打刻した状態で、同じ日に重ねて「出勤」を打刻できない制御が機能することを確認。

### 2.3 勤怠修正申請および承認・却下テスト
* **修正申請の登録と通知**:
  * 一般従業員が過去の打刻漏れに対して「打刻修正申請」を作成した際、`attendance_adjustments` テーブルにステータス `PENDING` で登録され、対象店舗の店長のダッシュボードへ通知が表示されることを確認。
* **店長による承認操作**:
  * 店長が申請を「承認」した際、対象の `attendance` レコードの時間が申請通りに書き換わり、申請ステータスが `APPROVED` に変更されることを確認。
* **店長による却下操作**:
  * 店長が申請を「却下」した際、対象の `attendance` レコード時間は書き換わらず元の状態が維持され、申請ステータスが `REJECTED` に変更されることを確認。
* **他店舗データの操作制限**:
  * 別店舗の店長がURLの申請IDを直接変更して他店舗の申請を承認・却下しようとした場合、エラーが発生して操作が制限されることを確認。

### 2.4 通知および画面遷移の検証
* **結果通知の作成**:
  * 修正申請が処理された際、申請を行った従業員宛ての通知データが正しく生成されることを確認。
* **通知詳細リンクからの遷移**:
  * 従業員画面に表示された通知の「詳細」リンクをクリックした際、結果を確認できる「打刻修正履歴」画面へ正しく遷移できることを確認。

### 2.5 月次の勤怠確認テスト
* **月次勤怠画面の表示確認**:
  * 店長および人事の画面で、所属する従業員の勤務実績（実働時間、遅刻数、残業時間など）が正しく計算され、一覧で表示できることを確認。

## 3. 本番環境（デプロイ後）の動作確認項目

Renderへのデプロイ完了後、以下の手順に沿って本番動作の手動検証を行います。

1. **初期管理者のログイン確認**:
   * 環境変数で設定した `INITIAL_HR_EMAIL` / `INITIAL_HR_PASSWORD` を用いて、正しくログインできることを確認。
2. **デモアカウントを用いたロール別アクセス確認**:
   * `DEMO_ACCOUNTS_ENABLED=true` にて自動生成されたデモ用アカウント（別途共有）を用いてログインし、一般従業員と店長のメニュー表示・画面遷移が適切に切り替わるかを確認。
3. **主要フローの一巡テスト**:
   * 従業員での打刻 → 打刻時間に基づく遅刻判定の確認 → 修正申請の作成 → 店長での承認 → 従業員の勤怠への時間反映、の一連のフローが本番環境（Supabase PostgreSQL）上でエラーなく機能することを確認。

## 4. 既知の制限事項と今後追加したいテスト

* **通知機能のテスト拡充**:
  * 現在はシステム内通知（ダッシュボード通知）を主なテスト対象としており、メール送信キューへの登録までを確認しています。今後はメール送信機能と連携した配送テストの検証を追加する予定です。
* **有休関連機能の整理・動作確認・ドキュメント整備**:
  * 将来的な機能拡張として、有休の自動付与計算、有休取得時の勤怠・シフトへの連動動作に対する単体・結合テストの追加を予定しています。
