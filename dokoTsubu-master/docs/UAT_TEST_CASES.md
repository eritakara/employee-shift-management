# ShiftFlow 受入テストケース

## 実施条件

- 対象: リリース候補版のShiftFlow
- 役割: 従業員、店長、人事担当者、店長の代理者
- 端末: 対応PCブラウザ、iPhone標準ブラウザ、Android Chrome
- 証跡: 実施日時、実施者、環境、結果、画面キャプチャまたは出力ファイル、不具合番号を各ケースに記録する
- 判定: 期待結果をすべて満たした場合だけ合格とする

## 主要受入条件17項目

| ID | 受入条件 | 主な手順 | 期待結果 | 自動試験・証跡の手掛かり |
|---|---|---|---|---|
| UAT-AC-01 | 翌月希望シフト提出と未提出通知 | 従業員Aは期限内に翌月希望を提出し、従業員Bは未提出のまま14日の定期処理を実行する。 | Aの希望が提出済みとなる。通知はBだけに一度送られ、Aには送られない。 | `CoreWorkflowE2ETest`、`NotificationRoutingTest`、通知一覧 |
| UAT-AC-02 | 店長による調整・警告・確定 | 店長が担当部署の勤務を編集し、人員不足を発生させて警告を確認後、理由を入力して月を確定する。 | 担当範囲だけ編集できる。人員不足警告が表示され、確定後は対象シフトが確定済みとなる。 | `ShiftWorkflowTest`、確定画面、監査ログ |
| UAT-AC-03 | 確定シフトの所属内公開 | 店長が月を確定し、同じ所属と別所属の従業員でシフト表を開く。 | 同じ営業所・部署の従業員は確定シフトを確認でき、権限外の所属データは表示されない。 | `CoreWorkflowE2ETest`、`SecurityAuthorizationTest` |
| UAT-AC-04 | 夜勤13時間と休息警告 | 17時から翌8時、休憩120分の夜勤を登録して勤務時間を確認し、翌日に日勤を登録する。 | 夜勤は開始日勤務・実働13時間として集計され、翌日勤務には夜勤明け休息警告が出る。 | `AttendanceCalculatorTest`、`ShiftWorkflowTest` |
| UAT-AC-05 | 日・半日・時間有休と代理承認 | 従業員が全日、午前半休、午後半休、1時間有休を申請する。店長と有効期間中の代理者がそれぞれ処理する。 | 各単位で申請でき、担当店長または有効な代理者だけが承認・却下できる。 | `LeavePolicyTest`、`CoreWorkflowE2ETest`、申請履歴 |
| UAT-AC-06 | 有休残数・時間上限 | 残数内の有休を承認し、残数超過と年間時間上限超過も申請する。 | 承認分だけ残数が減る。残数不足と時間単位上限超過は拒否され、取消時は残数が戻る。 | `LeavePolicyTest`、有休残数・履歴画面 |
| UAT-AC-07 | PC・スマートフォン打刻と位置情報 | PCとスマートフォンで出勤・退勤を行い、位置情報取得成功時の記録を確認する。 | 打刻時刻、緯度、経度、取得状態が対象従業員・日付に記録される。 | `CoreWorkflowE2ETest`、端末別画面キャプチャ、DB確認票 |
| UAT-AC-08 | 遅刻・早退・漏れ・残業表示 | 確定シフトに対し、遅刻、早退、退勤漏れ、所定時間超過となる打刻を各1件作成する。 | 各判定と残業時間が勤務区分の開始・終了・休憩設定に基づいて正しく表示される。 | `AttendanceCalculatorTest`、勤怠一覧 |
| UAT-AC-09 | 打刻修正と変更履歴 | 従業員が打刻修正を申請し、店長が承認する。承認前後の勤怠と監査ログを確認する。 | 承認後だけ修正値が反映され、申請値、処理者、処理日時、変更前後を追跡できる。 | `CoreWorkflowE2ETest`、`AttendanceAdjustmentAuditTest`、修正申請一覧、監査ログ |
| UAT-AC-10 | 店長月次確定と人事全社確認 | 店長が担当従業員の月次勤怠を確定し、人事担当者で複数所属の同月勤怠を開く。 | 店長は担当範囲を確定でき、人事担当者は全社分の確定状態と勤怠を確認できる。 | `AttendanceFinalizationTest`、月次確定画面 |
| UAT-AC-11 | 確定後ロックと解除 | 月次確定後に打刻・修正・修正承認を試し、店長が確定解除して再試行する。 | 確定中はすべて拒否され、解除後だけ修正と承認が可能になる。 | `AttendanceFinalizationTest` |
| UAT-AC-12 | 役割・所属別アクセス制御 | 従業員、店長、人事担当者、代理者で他者IDや別所属IDを指定し、一覧・更新・出力を試す。 | 権限外の閲覧・変更・出力が拒否され、画面からも権限外操作が提示されない。 | `SecurityAuthorizationTest`、`SecurityHardeningTest` |
| UAT-AC-13 | 役割別ダッシュボード | 各役割で、今日の勤務者、未承認、有休残数、人員不足、月間勤務時間を既知データと比較する。 | 5指標が正しく、従業員は本人、店長は担当所属、人事は全社の範囲で集計される。 | `SmokeTest`、ダッシュボード画面 |
| UAT-AC-14 | 月別グラフ | 6か月分の通常勤務、残業、有休を用意し、従業員・店長・人事でグラフを確認する。 | 月ごとの勤務時間、残業時間、有休取得数が元データと一致し、役割別範囲を越えない。 | グラフ画面キャプチャ、集計照合票 |
| UAT-AC-15 | Excel・CSV・印刷 | 各役割で許可された対象月をExcel・CSV出力し、月間シフトの印刷プレビューを開く。 | 文字化け・式注入がなく、列・行・対象範囲が画面と一致する。印刷時は操作部が除外される。 | `ExportServiceTest`、`SecurityHardeningTest`、出力ファイル |
| UAT-AC-16 | 日英・PC・スマートフォン | 言語を日本語と英語に切り替え、PCと各スマートフォンで主要業務を一巡する。 | 言語設定が維持され、文字切れ・重なり・操作不能がなく、全主要機能を完了できる。 | 端末・言語別チェックシート、画面キャプチャ |
| UAT-AC-17 | 5年間保存 | 保存期間境界の直前・当日・超過データをシフト、勤怠、有休、監査、位置情報に用意して保守処理を実行する。 | 5年以内は保持され、超過データは定義どおり削除または匿名化され、処理記録が残る。 | `DataRetentionServiceTest`、保守処理ログ |
