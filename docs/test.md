# テスト仕様・報告書

このドキュメントでは、シフト・勤怠管理アプリが正しく動くかを確認するために行ったテスト内容をまとめています。

どのような方針で確認したか、どの機能をテストしたか、本番環境へ公開した後に確認する項目について整理しています。

---

## 1. テスト方針

このアプリでは、できるだけシンプルな方法で品質を確認できるようにしています。

自動で基本的な動作を確認するスクリプト（`test.ps1`、`test-postgres.ps1`）を用意し、ログイン、打刻、申請、承認、権限チェックなど、重要な処理が問題なく動くかを確認しています。

### 主なテスト方法

* **自動テスト**
  * H2 Database などのテスト用データベースを使い、出退勤の打刻処理や権限チェックなど、アプリの重要な処理が正しく動くかを確認します。
* **ブラウザでの手動確認**
  * ローカル環境や本番環境で実際にブラウザを開き、画面が正しく表示されるか、操作の流れに問題がないかを確認します。

---

## 2. 確認した主な機能テスト項目

アプリの主要な機能について、以下の観点で動作確認を行っています。

### 2.1 ログインとアクセス制限の確認

* **ログインしていない状態でのアクセス制限**
  * ログインしていない状態で `/app/dashboard` や `/app/attendance/clock` などのURLへ直接アクセスした場合、ログイン画面（`/index.jsp`）へ戻されることを確認します。
* **権限のない画面へのアクセス拒否（403 Forbidden）**
  * 一般従業員アカウントでログインした状態で、店長専用画面（`/app/shifts/manage`、`/app/attendance/manage`）や人事専用画面（`/app/employees`）へ直接アクセスした場合、アクセスが拒否されることを確認します。
  * 店長アカウントで人事専用画面へアクセスした場合も、同じように制限が働くことを確認します。

### 2.2 出退勤打刻の確認

* **打刻時刻の記録**
  * 従業員アカウントで打刻画面を開き、「出勤」「退勤」ボタンを押したときに、現在時刻が勤怠データとして正しく登録されることを確認します。
* **遅刻・早退の自動判定**
  * 登録された出勤・退勤時刻と、あらかじめ登録されているシフトの時間を照らし合わせ、遅刻や早退が自動で判定されることを確認します。
* **二重打刻の防止**
  * すでに「出勤」を打刻している日に、もう一度「出勤」を打刻できないようになっていることを確認します。

### 2.3 勤怠修正申請と承認・却下の確認

* **修正申請の登録と通知**
  * 一般従業員が過去の打刻漏れに対して「打刻修正申請」を作成したときに、申請データが未処理の状態（`PENDING`）で登録されることを確認します。
  * あわせて、対象店舗の店長のダッシュボードに通知が表示されることを確認します。
* **店長による承認操作**
  * 店長が申請を「承認」したときに、対象の勤怠データの時間が申請内容どおりに更新され、申請ステータスが承認済み（`APPROVED`）になることを確認します。
* **店長による却下操作**
  * 店長が申請を「却下」したときに、対象の勤怠データは変更されず、申請ステータスが却下（`REJECTED`）になることを確認します。
* **他店舗データの操作制限**
  * 別店舗の店長がURLの申請IDを直接変更して、他店舗の申請を承認・却下しようとした場合、操作が制限されることを確認します。

### 2.4 通知と画面移動の確認

* **結果通知の作成**
  * 修正申請が承認または却下されたときに、申請した従業員へ結果通知が作成されることを確認します。
* **通知リンクからの画面移動**
  * 従業員画面に表示された通知の「詳細」リンクを押したときに、結果を確認できる「打刻修正履歴」画面へ移動できることを確認します。

### 2.5 月次の勤怠確認

* **月次勤怠画面の表示確認**
  * 店長や人事の画面で、従業員ごとの勤務実績（実働時間、遅刻回数、残業時間など）が正しく計算され、一覧で確認できることを確認します。

## 3. 本番環境での動作確認項目

Renderへデプロイした後、本番環境でも以下の流れを確認します。

1. **初期管理者のログイン確認**
   * 環境変数で設定した `INITIAL_HR_EMAIL` と `INITIAL_HR_PASSWORD` を使い、初期管理者として正しくログインできることを確認します。
2. **デモアカウントでの権限別確認**
   * `DEMO_ACCOUNTS_ENABLED=true` によって自動作成されたデモ用アカウントを使い、一般従業員・店長・人事でログインしたときに、表示されるメニューや使える画面が正しく切り替わることを確認します。
3. **主要な操作の一連確認**
   * 従業員で出退勤を打刻する。
   * 打刻時間に基づいて遅刻判定が行われることを確認する。
   * 従業員が打刻修正申請を作成する。
   * 店長が申請を承認する。
   * 承認後、従業員の勤怠データに修正後の時間が反映されることを確認する。

この一連の流れが、本番環境（Supabase PostgreSQL）上でエラーなく動くことを確認します。

## 4. 既知の制限事項と今後追加したいテスト

* **通知機能のテスト拡充**
  * 現在は、アプリ内の通知（ダッシュボード通知）を中心に確認しています。
  * 今後は、メール送信機能と連携した通知が正しく送られるかについても、さらに確認を追加する予定です。
* **有休関連機能の整理・動作確認・ドキュメント整備**
  * 将来的な機能拡張として、有休の自動付与や、有休取得時にシフト・勤怠へ正しく反映されるかを確認するテストを追加する予定です。
