# データベース構造定義書 (ShiftFlow)

本アプリケーションの起動時に `Database.java` によって自動生成されるリレーショナルデータベース（H2 / PostgreSQL 互換）のテーブル構造、カラム定義、および初期データの定義について解説します。

---

## 1. テーブル一覧とスキーマ定義

すべてのテーブルの主キー (`id`) には、PostgreSQL および H2 で互換性のある `BIGINT GENERATED BY DEFAULT AS IDENTITY` が使用されています。

### 1. 組織・マスタ関連

#### `branches` (店舗・拠点)
- `id` BIGINT PRIMARY KEY (自動採番)
- `name` VARCHAR(100) NOT NULL UNIQUE (拠点名)
- `active` BOOLEAN NOT NULL DEFAULT TRUE (有効状態)

#### `departments` (部署)
- `id` BIGINT PRIMARY KEY (自動採番)
- `name` VARCHAR(100) NOT NULL UNIQUE (部署名)
- `active` BOOLEAN NOT NULL DEFAULT TRUE (有効状態)

#### `employment_types` (雇用形態)
- `id` BIGINT PRIMARY KEY (自動採番)
- `name` VARCHAR(50) NOT NULL UNIQUE (雇用形態名)
- `active` BOOLEAN NOT NULL DEFAULT TRUE (有効状態)

#### `qualification_types` (保有資格定義)
- `id` BIGINT PRIMARY KEY (自動採番)
- `name` VARCHAR(100) NOT NULL UNIQUE (資格名)
- `active` BOOLEAN NOT NULL DEFAULT TRUE (有効状態)

#### `work_types` (勤務タイプ定義)
- `id` BIGINT PRIMARY KEY (自動採番)
- `code` VARCHAR(30) NOT NULL UNIQUE (勤務コード、例: DAY, NIGHT)
- `name_ja` VARCHAR(50) NOT NULL (日本語表示名)
- `name_en` VARCHAR(50) NOT NULL (英語表示名)
- `start_time` TIME (勤務開始時間)
- `end_time` TIME (勤務終了時間)
- `crosses_midnight` BOOLEAN NOT NULL DEFAULT FALSE (日跨ぎフラグ)
- `break_minutes` INT NOT NULL DEFAULT 0 (休憩時間)
- `required_staff` INT NOT NULL DEFAULT 0 (必要人数目安)
- `active` BOOLEAN NOT NULL DEFAULT TRUE (有効状態)

---

### 2. ユーザー関連

#### `users` (ユーザー情報)
- `id` BIGINT PRIMARY KEY (自動採番)
- `employee_number` VARCHAR(30) NOT NULL UNIQUE (社員番号)
- `name` VARCHAR(100) NOT NULL (氏名)
- `email` VARCHAR(200) NOT NULL UNIQUE (メールアドレス)
- `password_hash` VARCHAR(500) NOT NULL (パスワードハッシュ値)
- `hire_date` DATE NOT NULL (入社日)
- `branch_id` BIGINT NOT NULL (拠点ID、外部キー `branches(id)`)
- `department_id` BIGINT NOT NULL (部署ID、外部キー `departments(id)`)
- `employment_type_id` BIGINT NOT NULL (雇用形態ID、外部キー `employment_types(id)`)
- `role` VARCHAR(20) NOT NULL (権限ロール: `HR`, `MANAGER`, `EMPLOYEE`)
- `locale` VARCHAR(5) NOT NULL DEFAULT 'ja' (使用言語)
- `active` BOOLEAN NOT NULL DEFAULT TRUE (有効状態)
- `weekly_work_days` INT NOT NULL DEFAULT 5 (週労働日数)
- `weekly_work_hours` DECIMAL(5,2) NOT NULL DEFAULT 40 (週労働時間数)
- `deactivated_at` TIMESTAMP (無効化日時)

#### `qualifications` (保有資格一覧)
- `id` BIGINT PRIMARY KEY (自動採番)
- `user_id` BIGINT NOT NULL (ユーザーID、外部キー `users(id)`)
- `name` VARCHAR(100) NOT NULL (資格名)
- `expires_on` DATE (有効期限)
- `active` BOOLEAN NOT NULL DEFAULT TRUE (有効状態)

---

### 3. シフト関連

#### `shifts` (確定シフト)
- `id` BIGINT PRIMARY KEY (自動採番)
- `user_id` BIGINT NOT NULL (ユーザーID、外部キー `users(id)`)
- `work_date` DATE NOT NULL (勤務日)
- `work_type_code` VARCHAR(30) NOT NULL (勤務タイプコード、外部キー `work_types(code)`)
- `status` VARCHAR(20) NOT NULL DEFAULT 'DRAFT' (状態: `DRAFT`, `CONFIRMED`)
- `note` VARCHAR(500) (メモ)
- `updated_by` BIGINT NOT NULL (更新者ユーザーID、外部キー `users(id)`)
- `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (更新日時)
- *複合ユニーク制約*: `(user_id, work_date)`

#### `shift_preference_submissions` (希望シフト提出状況)
- `id` BIGINT PRIMARY KEY (自動採番)
- `user_id` BIGINT NOT NULL (ユーザーID、外部キー `users(id)`)
- `target_month` DATE NOT NULL (対象年月)
- `status` VARCHAR(20) NOT NULL DEFAULT 'DRAFT' (提出状態: `DRAFT`, `SUBMITTED`, `APPROVED`)
- `submitted_at` TIMESTAMP (提出日時)
- `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (更新日時)
- `reviewed_by` BIGINT (承認者、外部キー `users(id)`)
- `reviewed_at` TIMESTAMP (承認日時)
- *複合ユニーク制約*: `(user_id, target_month)`

#### `shift_preferences` (希望シフト明細)
- `id` BIGINT PRIMARY KEY (自動採番)
- `submission_id` BIGINT NOT NULL (提出ID、外部キー `shift_preference_submissions(id)` ON DELETE CASCADE)
- `preference_date` DATE NOT NULL (希望日)
- `request_type` VARCHAR(20) NOT NULL (希望勤務形態/休暇要求)
- `note` VARCHAR(500) (特記事項)
- *複合ユニーク制約*: `(submission_id, preference_date)`

#### `shift_change_requests` (シフト変更申請)
- `id` BIGINT PRIMARY KEY (自動採番)
- `user_id` BIGINT NOT NULL (申請者、外部キー `users(id)`)
- `work_date` DATE NOT NULL (変更対象日)
- `requested_work_type` VARCHAR(30) NOT NULL (希望する勤務コード)
- `reason` VARCHAR(1000) NOT NULL (申請理由)
- `urgent` BOOLEAN NOT NULL DEFAULT FALSE (緊急フラグ)
- `status` VARCHAR(20) NOT NULL DEFAULT 'PENDING' (承認状態: `PENDING`, `APPROVED`, `REJECTED`)
- `decided_by` BIGINT (判定者、外部キー `users(id)`)
- `decided_at` TIMESTAMP (判定日時)
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (申請作成日時)

### 4. 有給休暇関連

#### `leave_balances` (有給休暇残高)
- `user_id` BIGINT PRIMARY KEY (ユーザーID、外部キー `users(id)`)
- `days_remaining` DECIMAL(6,2) NOT NULL DEFAULT 10 (残日数)
- `hourly_used` INT NOT NULL DEFAULT 0 (時間休消費時間数)
- `last_granted_on` DATE (最終付与日)

#### `leave_rule_config` (有休管理ルール設定)
- `id` BIGINT PRIMARY KEY (自動採番)
- `effective_from` DATE NOT NULL UNIQUE (適用開始日)
- `attendance_threshold` DECIMAL(4,3) NOT NULL DEFAULT 0.800 (出勤率閾値、標準80%)
- `hourly_limit_days` INT NOT NULL DEFAULT 5 (時間休上限日数)
- `hours_per_day` INT NOT NULL DEFAULT 8 (1日あたり標準労働時間、時間休換算用)
- `expiry_months` INT NOT NULL DEFAULT 24 (有休有効期限月数、標準2年間)
- `mandatory_days` INT NOT NULL DEFAULT 5 (義務化年5日出勤ルール)
- `active` BOOLEAN NOT NULL DEFAULT TRUE (有効状態)

#### `leave_grants` (有給付与履歴)
- `id` BIGINT PRIMARY KEY (自動採番)
- `user_id` BIGINT NOT NULL (ユーザーID、外部キー `users(id)`)
- `grant_date` DATE NOT NULL (付与日)
- `expires_on` DATE NOT NULL (有効期限)
- `days_granted` DECIMAL(6,2) NOT NULL (付与日数)
- `days_remaining` DECIMAL(6,2) NOT NULL (付与分の残日数)
- `attendance_rate` DECIMAL(5,4) (算定期間 of 出勤率)
- `source` VARCHAR(30) NOT NULL (付与元: `STATUTORY`(法定), `MIGRATION`(移行))
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (作成日時)
- *複合ユニーク制約*: `(user_id, grant_date)`

#### `leave_consumptions` (有休消化紐付け明細)
- `id` BIGINT PRIMARY KEY (自動採番)
- `request_id` BIGINT NOT NULL (有休申請ID)
- `grant_id` BIGINT NOT NULL (消費対象の付与履歴ID、外部キー `leave_grants(id)`)
- `days_used` DECIMAL(6,3) NOT NULL (消費日数)
- `hours_used` INT NOT NULL DEFAULT 0 (消費時間数)
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (消化日時)
- `restored` BOOLEAN NOT NULL DEFAULT FALSE (ロールバック復元フラグ)
- *複合ユニーク制約*: `(request_id, grant_id)`

#### `leave_history` (有休履歴ログ)
- `id` BIGINT PRIMARY KEY (自動採番)
- `user_id` BIGINT NOT NULL (ユーザーID、外部キー `users(id)`)
- `event_type` VARCHAR(30) NOT NULL (イベント: `GRANT`, `USE`, `EXPIRE`, `RESTORE`, `INELIGIBLE`)
- `event_date` DATE NOT NULL (発生日)
- `days` DECIMAL(6,3) NOT NULL DEFAULT 0 (増減日数)
- `hours` INT NOT NULL DEFAULT 0 (増減時間数)
- `request_id` BIGINT (関連有休申請ID)
- `grant_id` BIGINT (関連付与履歴ID)
- `note` VARCHAR(500) (補足説明)
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (作成日時)

#### `leave_requests` (有休申請)
- `id` BIGINT PRIMARY KEY (自動採番)
- `user_id` BIGINT NOT NULL (申請者、外部キー `users(id)`)
- `leave_date` DATE NOT NULL (休暇取得希望日)
- `leave_unit` VARCHAR(20) NOT NULL (取得単位: `FULL`, `AM`, `PM`, `HOURLY`)
- `hours` INT (時間単位有休時の時間数)
- `reason` VARCHAR(1000) NOT NULL (申請理由)
- `status` VARCHAR(20) NOT NULL DEFAULT 'PENDING' (状態: `PENDING`, `APPROVED`, `REJECTED`)
- `decided_by` BIGINT (判定者、外部キー `users(id)`)
- `decided_at` TIMESTAMP (判定日時)
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (申請日時)

### 5. 勤怠関連

#### `attendance` (出退勤記録)
- `id` BIGINT PRIMARY KEY (自動採番)
- `user_id` BIGINT NOT NULL (ユーザーID、外部キー `users(id)`)
- `work_date` DATE NOT NULL (打刻日)
- `clock_in` TIMESTAMP (出勤打刻日時)
- `clock_out` TIMESTAMP (退勤打刻日時)
- `in_lat` DECIMAL(10,7) (出勤打刻時の緯度)
- `in_lng` DECIMAL(10,7) (出勤打刻時の経度)
- `out_lat` DECIMAL(10,7) (退勤打刻時の緯度)
- `out_lng` DECIMAL(10,7) (退勤打刻時の経度)
- `location_status` VARCHAR(30) (位置情報の有効判定)
- `status` VARCHAR(20) NOT NULL DEFAULT 'OPEN' (状態: `OPEN`, `COMPLETE`, `ADJUSTED`)
- `finalized` BOOLEAN NOT NULL DEFAULT FALSE (締め処理完了フラグ)
- *複合ユニーク制約*: `(user_id, work_date)`

#### `attendance_adjustments` (打刻修正申請)
- `id` BIGINT PRIMARY KEY (自動採番)
- `attendance_id` BIGINT NOT NULL (対象打刻データID、外部キー `attendance(id)`)
- `requested_by` BIGINT NOT NULL (申請者、外部キー `users(id)`)
- `requested_in` TIMESTAMP (修正希望の出勤日時)
- `requested_out` TIMESTAMP (修正希望の退勤日時)
- `reason` VARCHAR(1000) NOT NULL (修正理由)
- `status` VARCHAR(20) NOT NULL DEFAULT 'PENDING' (承認状態: `PENDING`, `APPROVED`, `REJECTED`)
- `decided_by` BIGINT (判定者、外部キー `users(id)`)
- `decided_at` TIMESTAMP (判定日時)
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (修正申請日時)

### 6. システム・その他関連

#### `company_holidays` (公休日・会社休日)
- `id` BIGINT PRIMARY KEY (自動採番)
- `holiday_date` DATE NOT NULL (休日日付)
- `branch_id` BIGINT (拠点ID、外部キー `branches(id)`、NULLの場合は全社共通休日)
- `name` VARCHAR(100) NOT NULL (休日名称、例: 年末年始)
- *複合ユニーク制約*: `(holiday_date, branch_id)`

#### `delegations` (管理権限の移譲)
- `id` BIGINT PRIMARY KEY (自動採番)
- `manager_id` BIGINT NOT NULL (移譲元マネージャー、外部キー `users(id)`)
- `delegate_id` BIGINT NOT NULL (移譲先社員、外部キー `users(id)`)
- `starts_on` DATE NOT NULL (適用開始日)
- `ends_on` DATE NOT NULL (適用終了日)
- `active` BOOLEAN NOT NULL DEFAULT TRUE (有効状態)

#### `app_settings` (システム設定値キーバリュー)
- `setting_key` VARCHAR(80) PRIMARY KEY (設定キー名)
- `setting_value` VARCHAR(500) NOT NULL (設定値)
- `description` VARCHAR(500) (設定内容の説明文)
- `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (更新日時)

#### `notifications` (システム内通知)
- `id` BIGINT PRIMARY KEY (自動採番)
- `user_id` BIGINT NOT NULL (通知先、外部キー `users(id)`)
- `type` VARCHAR(50) NOT NULL (通知種別)
- `title` VARCHAR(200) NOT NULL (通知タイトル)
- `message` VARCHAR(1000) NOT NULL (通知メッセージ本文)
- `target_url` VARCHAR(500) (遷移先URL)
- `is_read` BOOLEAN NOT NULL DEFAULT FALSE (既読フラグ)
- `email_status` VARCHAR(20) NOT NULL DEFAULT 'QUEUED' (メール送信状況)
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (作成日時)

#### `account_tokens` (トークン認証用)
- `id` BIGINT PRIMARY KEY (自動採番)
- `user_id` BIGINT NOT NULL (ユーザーID、外部キー `users(id)`)
- `token` VARCHAR(100) NOT NULL UNIQUE (トークン文字列)
- `token_type` VARCHAR(20) NOT NULL (トークン種別: パスワードリセット等)
- `expires_at` TIMESTAMP NOT NULL (有効期限)
- `used_at` TIMESTAMP (使用済み日時)
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (作成日時)

#### `mail_outbox` (システムメール送信キュー)
- `id` BIGINT PRIMARY KEY (自動採番)
- `recipient` VARCHAR(200) NOT NULL (宛先メールアドレス)
- `subject` VARCHAR(300) NOT NULL (件名)
- `body` VARCHAR(4000) NOT NULL (本文)
- `status` VARCHAR(20) NOT NULL DEFAULT 'QUEUED' (送信状態: `QUEUED`, `SENT`, `FAILED`)
- `attempts` INT NOT NULL DEFAULT 0 (再試行回数)
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (キュー作成日時)
- `sent_at` TIMESTAMP (送信完了日時)
- `last_error` VARCHAR(2000) (最終エラーメッセージ)
- `next_attempt_at` TIMESTAMP (次回送信試行日時)

#### `audit_logs` (変更監査ログ)
- `id` BIGINT PRIMARY KEY (自動採番)
- `actor_id` BIGINT (操作者ユーザーID、外部キー `users(id)`)
- `action` VARCHAR(80) NOT NULL (操作アクション名)
- `target_type` VARCHAR(80) NOT NULL (対象オブジェクト種別)
- `target_id` VARCHAR(80) (対象オブジェクトのID値)
- `before_value` TEXT (変更前のJSON/テキストデータ)
- `after_value` TEXT (変更後のJSON/テキストデータ)
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP (操作実行日時)
- `target_user_id` BIGINT (変更対象となったユーザーID)

## 2. 初期シードデータ定義

データベースが空の状態で起動された場合、`Database.java` の `seed` メソッドによって以下の既定レコードが自動登録されます。

### 1. 組織マスタデータ
- **拠点 (`branches`)**: `本社`, `北部支店`, `中部支店`, `那覇支店`, `南部支店`, `宮古支店`, `石垣支店`
- **部署 (`departments`)**: `営業部`, `経理部`, `総務部`, `人事部`
- **雇用形態 (`employment_types`)**: `正社員`, `契約社員`, `パート`, `アルバイト`

### 2. シフト定義 (`work_types`)
| コード | 表示名 (日) | 表示名 (英) | 開始時間 | 終了時間 | 日またぎ | 休憩 (分) | 必要人数 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `DAY` | 日勤 | Day | 08:00 | 17:00 | なし | 60 | 5 |
| `NIGHT` | 夜勤 | Night | 17:00 | 08:00 (翌) | あり | 120 | 7 |

- **適用開始日**: 2019-04-01

- `LEAVE_MIN_NOTICE_DAYS` = `1` (有休申請の最低事前日数)
- `MONTHLY_CLOSE_DAY` = `5` (翌月の勤怠締切日)
- `RETENTION_YEARS` = `5` (業務データ保存年数)
- `LOCATION_REQUIRED` = `false` (位置情報なしの打刻を禁止)
- `MAX_CONCURRENT_USERS` = `100` (想定最大同時利用者数)
